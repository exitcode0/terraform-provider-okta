# Resource: okta_entity_risk_policy_rule


Manages an Entity Risk Policy Rule. Entity Risk Policy rules define automated responses to identity threats detected by Okta's Identity Threat Protection (ITP).


## Links

- [Okta API docs](https://developer.okta.com/docs/api/openapi/okta-management/management/tag/Policy/)
- [Provider source](https://github.com/okta/terraform-provider-okta/blob/master/okta/services/idaas/resource_okta_entity_risk_policy_rule.go)
- [SDK source](https://github.com/okta/okta-sdk-golang/blob/v6.1.6/okta/api_policy.go)

## Example Usage

```terraform
data "okta_entity_risk_policy" "example" {
}

resource "okta_entity_risk_policy_rule" "example" {
  policy_id              = data.okta_entity_risk_policy.example.id
  name                   = "High Risk Response"
  risk_level             = "HIGH"
  terminate_all_sessions = true
}
```

## Schema

### Required

- `name` (String) Name of the policy rule.
- `policy_id` (String) ID of the Entity Risk Policy. Use the `okta_entity_risk_policy` data source to get this ID.
- `risk_level` (String) Risk level to match. Valid values: HIGH, MEDIUM, LOW, ANY.

### Optional

- `groups_excluded` (Set of String) List of group IDs to exclude from this rule.
- `groups_included` (Set of String) List of group IDs to include in this rule.
- `priority` (Number) Priority of the rule. Rules are evaluated in priority order.
- `status` (String) Status of the rule: ACTIVE or INACTIVE.
- `terminate_all_sessions` (Boolean) When true, terminates all active sessions for the user when a risk event is detected.
- `users_excluded` (Set of String) List of user IDs to exclude from this rule.
- `users_included` (Set of String) List of user IDs to include from this rule.
- `workflow_id` (String) ID of the Okta Workflow to run when a risk event is detected.

### Read-Only

- `id` (String) ID of the policy rule.
